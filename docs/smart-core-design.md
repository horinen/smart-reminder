# 智能核心设计 - 定时任务2

定时任务2是系统中唯一需要AI智能判断的环节，负责统合所有信息，生成和调整提醒规则。

## 设计约束

**Agent 不能直接读写文件**，所有数据操作必须通过脚本执行：
- 读操作：通过脚本获取数据
- 写操作：通过脚本更新数据

脚本层是 Agent 与数据之间的桥梁。

---

```

│                                                             │
```

## 脚本设计

### 读取类脚本

**输出格式**：纯文本摘要 + Markdown轻量结构

**设计原则**：
- 脚本预处理计算字段（如"距上次更新天数"），AI直接使用
- 不需要时间范围参数（历史数据定期清理）

#### get-projects.mjs

**职责**：获取项目信息

**输出示例**：
```markdown
## 项目状态
- 项目A：活跃，2天前更新
- 项目B：暂停，5天前更新
- 项目C：活跃，1天前更新
```

#### get-schedules.mjs

**职责**：获取用户日程信息

**输出示例**：
```markdown
## 近期日程
- 2026-03-20 14:00-16:00：会议
- 2026-03-21 全天：空闲
```

#### get-reminder-history.mjs

**职责**：获取提醒历史和效果反馈

**输出示例**：
```markdown
## 提醒效果（近7天）
- 发送5条，响应4条
- 平均响应时间：15分钟
- 正面反馈：2条，负面反馈：1条
```

### 写入类脚本

#### add-pending-reminder.mjs

**职责**：新增临时提醒

**参数**：
```bash
--time "2026-03-20 20:00"
--content "项目A已经2天没更新了，要继续吗？"
```

#### add-recurring-reminder.mjs

**职责**：新增持久提醒

**参数**：
```bash
--id "recurring-001"
--time "2026-03-21 20:00"
--content "每周回顾项目进度"
```

#### delete-recurring-reminder.mjs

**职责**：删除持久提醒

**参数**：
```bash
--id "recurring-001"
```

## 规则格式设计（任务1与任务2的契约）

定时任务1需要"确定性判断"，因此任务2生成的规则必须是**机器可解析的结构化格式**。

### 一次性提醒格式（pending-reminders.json）

```json
{"id": "xxx", "time": "2026-03-20 20:00", "content": "项目A已经2天没更新了，要继续吗？"}
```

**字段说明**：
- `id`：唯一标识
- `time`：提醒时间（精确到分钟）
- `content`：提醒内容（任务1直接发送）

**特点**：发送后删除

### 可重复提醒格式（recurring-reminders.json）

**格式**：与一次性提醒一致

```json
{"id": "recurring-001", "time": "2026-03-21 20:00", "content": "每周回顾项目进度"}
```

**区别**：
- 存放在不同文件
- 持久存在，由AI手动删除

## AI处理设计

### 执行时机

- **频率**：每天2次
- **时间点**：早9:00、晚20:00

### 调整依据

- **显式反馈**：用户说"这个时间好/不好"
- **隐式行为**：响应时间、忽略率、正面/负面反馈

### 智能判断原则

AI发挥主观能动性，根据以下指导原则灵活判断：

**判断项目是否需要提醒**：
- 距上次更新时间
- 项目重要性/优先级
- 项目状态（暂停的可能不需要）
- 用户近期日程（忙的时候少打扰）

**判断提醒时机**：
- 用户空闲时段
- 历史提醒效果（什么时间响应率高）
- 避免频繁打扰

### 边界约束

AI判断时必须遵守：
- 不在 22:00-08:00 提醒
- 同一项目 24 小时内不重复提醒

### 分析流程

1. **识别需要关注的项目**：综合时间、状态、重要性判断
2. **判断最佳提醒时机**：结合用户日程、历史效果
3. **生成提醒**：
   - 临时提醒：写入 pending-reminders.json
   - 持久提醒：写入 recurring-reminders.json
4. **调整提醒**：根据反馈删除或更新持久提醒

### 指令存储

| 位置 | 内容 |
|------|------|
| AGENTS.md | 指导原则、边界约束（AI如何判断） |
| Cron message | 任务步骤（AI要做什么） |

**Cron message 示例**：
```
执行定时任务2：读取项目、日程、历史数据，分析后生成或调整提醒。
```
